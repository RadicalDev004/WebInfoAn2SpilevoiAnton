Specificații Funcționale - Aplicație de Gestiune a Cărților

Autori: Șpilevoi Bogdan, Anton Cosmin

Data: 3 iunie 2025

Rezumat

Aplicația web dezvoltată urmează arhitectura MVC și oferă utilizatorilor posibilitatea de a se autentifica, vizualiza, urmări progresul, adăuga și salva cărți în lista lor de favorite. Acest document descrie funcționalitățile esențiale ale sistemului și interacțiunea cu utilizatorul, conform șablonului IEEE SRS.

1. Introducere

Aplicația are ca scop oferirea unei platforme web prin care utilizatorii pot gestiona o colecție personalizată de cărți. Utilizatorii se pot autentifica, naviga printre cărți, salva favorite și accesa setări personale.

2. Funcționalități esențiale

  • 2.1 Autentificare: utilizatorii pot crea/loga cu conturile lor (controller: controllerauth.php, model: modelauth.php, view: auth.tpl).
  • 2.2 Vizualizare cărți: lista cărților disponibile poate fi explorată (controller: controllerbook.php, model: modelbook.php, view: book.tpl).
  • 2.3 Vizualizare cărți externe: aplicația conține și cărti provenite din serviciul API extern GoogleBooks.
  • 2.4 Căutare cărți: aplicația permite căutarea de cărți dupa mai multe criterii (titlu, autor, editură, an) prin ambele biblioteci (internă, externă).
  • 2.5 Favorite: utilizatorii pot adăuga cărți în lista de favorite (favorite.php).
  • 2.6 Citire carte: utilizatorul poate "citi" pagini din fiecare carte, aplicația ținând minte progresul, fie că este o carte din librăria locala sau externă.
  • 2.7 Oferire recenzie: utilizatorii pot lasa recenzii la carti, cu un maxim de 5 stele plus un text descriptiv al experientei.
  • 2.8 Rss: aplicația ofera un flux RSS cu ultimele informatii despre: Cel mai recent top 10 carți, Top 25 cele mai recente recenzii.
  • 2.9 Setări cont: Vizualizarea datelor personale se realizează printr-un modul separat (controllersettings.php, modelsettings.php, settings.tpl).
  • 2.10 Vizualizare/exportare a datelor contului: utilizatorul poate viziona o versiune succintă a datelor sale în aplicație (setări) sau poate downloada in format CSV o versiune extensivă.

3. Interacțiunea cu utilizatorul

Interfața este împărțită în mai multe componente (views) care se încarcă în funcție de acțiunea utilizatorului. Navigarea se face prin linkuri în bara de meniu, iar interacțiunile (ex: adăugarea la favorite) sunt procesate prin request-uri către api-ul PHP.

  • 3.1 Formular de autentificare/logare: permite logarea și crearea unui cont nou.
  • 3.2 Pagina de start: (controllerhome.php, home.tpl) oferă acces rapid la cărți, favorite, neterminate, top cărți și rss.
  • 3.3 Pagina de setări: permite de-logarea și vizualizarea/exportarea datelor contului.
  • 3.4 Pagina de vizualizare a unei cărți: permite interacțiunea cu o anumită carte.
  • 3.5 Pagina de admin: permite administrarea datelor din baza de date a aplicației.
  • 3.6 Pagini dinamice: fiecare acțiune trimite date către un controller, care apelează un model și apoi încarcă o pagină .tpl.

4. Arhitectură tehnică

4.1 Proiectul respectă principiile MVC:

  • Model: manipularea datelor (fișierele model*.php).
  • View: afișarea datelor HTML în fișierele *.tpl.
  • Controller: logica de legătură între view și model (controller*.php).

4.2 Proiectul apelează API-uri proprii:

  • /api/favorite adăugarea unei cărți la favorite (fișierulfavorite.php).
  • /api/read adăugarea de pagini la progresul unei cărți. (fișierulread.php).
  • /api/review adăugarea unei recenzii la o carte, (fișierulreview.php).
  • /api/rss facilitarea fluxului rss. (fișierulrss.php).
  • /api/addentry adăugarea unei linii in o tabelă din baza de date. (fișieruladdentry.php).
  • /api/deleteentry ștergerea unei linii in o tabelă din baza de date. (fișieruldeleteentry.php).

5. Cerințe nefuncționale

  • 5.1 Performanță: Aplicația va răspunde în maximum 2 secunde la orice solicitare utilizator autenticat, inclusiv căutări sau accesări de cărți.
  • 5.2 Securitate: Datele utilizatorului (autentificare, favorite, recenzii) sunt protejate prin sesiuni PHP, criptare parole și validare server-side.
  • 5.3 Portabilitate: Aplicația este compatibilă cu browsere moderne (Chrome, Firefox, Edge) și poate fi rulată local sau pe servere de tip Apache/Nginx.
  • 5.4 Extensibilitate: Structura modulară MVC permite adăugarea facilă de noi funcționalități și API-uri fără a afecta componentele existente.
  • 5.5 Disponibilitate: Sistemul trebuie să fie accesibil minim 99% din timp într-un mediu de producție.

6. Glosar de termeni

MVC
Model-View-Controller, o arhitectură software care separă logica de business (model), interfața utilizator (view) și logica aplicației (controller).
API
Interfață de programare a aplicației care permite comunicația între diferite componente software.
RSS
Really Simple Syndication – tehnologie care oferă un flux actualizat de conținut, cum ar fi știri sau recenzii.
CSV
Comma-Separated Values – format text utilizat pentru stocarea datelor tabulare în fișiere.
Recenzie
Evaluare scrisă și/sau notată a unei cărți de către utilizator, reflectând experiența sa de lectură.

7. Concluzie

Aplicația oferă o interfață simplă și intuitivă pentru gestionarea cărților favorite, respectând o arhitectură clară și extensibilă. Specificațiile prezentate permit dezvoltarea și testarea coerentă a aplicației.
